# <a name="import-or-manually-create-postal-codes"></a>Importez ou créez manuellement les codes postaux

Cet article décrit la procédure d'importation et créer manuellement les codes postaux dans le format approprié. Cette rubrique fournit des informations sur la fonction qui a été ajoutée pour Microsoft Dynamics 365 pour les opérations. 

Le processus d'importation permet de mettre les codes postaux à jour d'un pays/une région spécifique. Vous pouvez également créer les codes postaux manuellement.

## <a name="import-zippostal-codes"></a>Importer des codes postaux
Vous pouvez utiliser ** codes postaux d'importation ** la page pour importer nouveaux codes postaux dans Microsoft Dynamics 365 pour les opérations. Lorsque vous importez les codes, la s'il ou les codes postaux existants sont remplacés par le nouveau format, et les nouveaux codes sont ajoutés.

Pour certains pays, vous devez utiliser le cadre de gestion des données permet d'importer des codes, tandis que d'autres pays un seul fichier de chargement est requis. Belgique, les Pays-Bas, et la Suède exigent un fichier pour les télécharger.

> [!NOTE]
> -   Pour la Belgique, le page Web officiel du valider belge fournit une cote officielle les codes postaux et des noms correspondants de ville. Le format de fichier HTML d'applications d'assistance à l'importation.
> -   Pour les Pays-Bas, une tiers organisation fournit le fichier qui contient les codes postaux. Après l'importation soit complétée, tous les codes postaux figurent dans le format (NNNN AA.)
> -   Pour la Suède, Postnummerservice.se propose deux types de fichiers : Codes postaux et adresses suédois de Suédois. Le format de fichier texte d'aide à l'importation pour les deux types.


## <a name="create-zippostal-codes-manually"></a>Création manuelle des codes postaux
Au lieu d'importer des codes, vous pouvez utiliser ** adresse paramétrée ** la page permet d'ajouter manuellement nouveaux codes postaux.
