# <a name="generate-er-reports-by-adding-content-as-raw-xml"></a><span data-ttu-id="8015a-103">Générer des états ER en ajoutant du contenu au format XML brut</span><span class="sxs-lookup"><span data-stu-id="8015a-103">Generate ER reports by adding content as raw XML</span></span>

[!include[banner](../includes/banner.md)]

<span data-ttu-id="8015a-104">Vous pouvez utiliser le nouvel élément de format **XML BRUT** pour créer des formats de gestion des états électroniques qui génèrent des documents sortants au format XML.</span><span class="sxs-lookup"><span data-stu-id="8015a-104">You can use the new **RAW XML** format element to design Electronic reporting (ER) formats that generate outgoing documents in XML format.</span></span> <span data-ttu-id="8015a-105">Dans certains cas, vous préférez peut-être ajouter des données XML brutes à ces états pour une ou plusieurs des raisons suivantes :</span><span class="sxs-lookup"><span data-stu-id="8015a-105">In some cases, you might prefer to add raw XML data to these reports for one or more of the following reasons:</span></span>

- <span data-ttu-id="8015a-106">Il est plus pratique d'utiliser le format XML brut pour la conception d'origine et la maintenance continue d'un état, car la structure XML peut être générée automatiquement en exécutant une expression au moment de l'exécution.</span><span class="sxs-lookup"><span data-stu-id="8015a-106">It's more convenient to use raw XML for the original design and ongoing maintenance of a report, because the XML structure can be automatically generated by executing a runtime expression.</span></span> <span data-ttu-id="8015a-107">Par conséquent, il n'est pas nécessaire de déterminer plusieurs liaisons pour plusieurs éléments de format au moment de la conception.</span><span class="sxs-lookup"><span data-stu-id="8015a-107">Therefore, multiple bindings don't have to be determined for multiple format elements at design time.</span></span> <span data-ttu-id="8015a-108">Cela est possible lorsque les sources de données que vous utilisez contiennent des informations permettant de créer des éléments XML lors de la génération de l'état.</span><span class="sxs-lookup"><span data-stu-id="8015a-108">It is possible when the data sources that you're using contain information that can be used to make XML elements while the report is generated.</span></span> 
- <span data-ttu-id="8015a-109">Aucune autre méthode ne peut être utilisée pour renseigner l'état avec le contenu XML qui a été précédemment reçu et enregistré dans le système.</span><span class="sxs-lookup"><span data-stu-id="8015a-109">No other method can be used to fill the report with XML content that was previously received and stored in the system.</span></span> <span data-ttu-id="8015a-110">Par exemple, la réponse XML générée doit contenir le contenu d'une demande XML qui a été précédemment envoyée.</span><span class="sxs-lookup"><span data-stu-id="8015a-110">For example, the XML response that is generated might have to contain the content of an XML request that was sent earlier.</span></span>
- <span data-ttu-id="8015a-111">Aucune autre méthode ne peut être utilisée pour insérer des caractères dans le document généré en fonction de leurs codes numériques.</span><span class="sxs-lookup"><span data-stu-id="8015a-111">No other method can be used to insert characters into the generated document based on their numeric codes.</span></span> <span data-ttu-id="8015a-112">Pour certaines langues et certains caractères, les codes de ce type n'existent pas.</span><span class="sxs-lookup"><span data-stu-id="8015a-112">For some languages and characters, codes of this type don't exist.</span></span> <span data-ttu-id="8015a-113">Il s'agit par exemple de la lettre grec rho (ρ) et des codes d'entité HTML tels que \&eacute; pour un *e* avec un accent aigu (é).</span><span class="sxs-lookup"><span data-stu-id="8015a-113">Examples include the Greek letter rho (ρ) and HTML entity codes such as \&eacute; for an *e* that has an acute accent (é).</span></span>

> [!NOTE]
> <span data-ttu-id="8015a-114">Notez que la structure ne contrôle pas si le contenu XML placé dans le document généré à l'aide de l'élément de format **XML BRUT** est correct.</span><span class="sxs-lookup"><span data-stu-id="8015a-114">Be aware that the framework doesn't control whether the XML content that is placed to the generated document by using the **RAW XML** format element is correct.</span></span>
