# <a name="wave-allocation"></a><span data-ttu-id="96463-103">Répartition des vagues</span><span class="sxs-lookup"><span data-stu-id="96463-103">Wave allocation</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="96463-104">Le traitement des vagues peut prendre du temps et la majeure partie du temps de traitement est consacrée à l’étape de répartition et à l’étape de création du travail.</span><span class="sxs-lookup"><span data-stu-id="96463-104">Wave processing can be time consuming, and most of the processing time is spent in the allocation step and in the work creation step.</span></span>

<span data-ttu-id="96463-105">Il est maintenant possible d’exécuter chacune de ces étapes en parallèle, ce qui peut améliorer les performances du traitement des vagues et permettre un plus grand débit de vagues dans le même entrepôt.</span><span class="sxs-lookup"><span data-stu-id="96463-105">It is now possible to run each of these steps in parallel, which can improve the performance of the wave processing, and allow for a larger throughput of waves in the same warehouse.</span></span> <span data-ttu-id="96463-106">Cette rubrique explique comment configurer la méthode de répartition des vagues pour qu’elle s’exécute en parallèle.</span><span class="sxs-lookup"><span data-stu-id="96463-106">This topic explains how to set up the wave allocation method to run in parallel.</span></span> <span data-ttu-id="96463-107">Pour plus d’informations sur la configuration de la création de travaux pour qu’elle s’exécute en parallèle, voir [Planifier la création du travail pendant la vague](configure-wave-schedule-work-creation.md).</span><span class="sxs-lookup"><span data-stu-id="96463-107">For more information about how to set up work creation to run in parallel, see [Schedule work creation during wave](configure-wave-schedule-work-creation.md).</span></span>

<span data-ttu-id="96463-108">Auparavant, il n’était possible d’allouer qu’une seule vague à la fois dans un entrepôt.</span><span class="sxs-lookup"><span data-stu-id="96463-108">Previously it was only possible to allocate one wave at a warehouse at a time.</span></span> <span data-ttu-id="96463-109">Cette contrainte a été supprimée et remplacée par une nouvelle contrainte qui verrouille uniquement l’article et les dimensions qui se trouvent au-dessus de l’emplacement dans la hiérarchie de réservation.</span><span class="sxs-lookup"><span data-stu-id="96463-109">This constraint has been removed and replaced by a new constraint that only locks the item and dimensions that are above location in the reservation hierarchy.</span></span> <span data-ttu-id="96463-110">Les dimensions au-dessus de l’emplacement incluent toujours les dimensions du produit.</span><span class="sxs-lookup"><span data-stu-id="96463-110">Dimensions above the location always include product dimensions.</span></span> <span data-ttu-id="96463-111">Par exemple, si un article est configuré à l’aide de la *Couleur*, alors les variantes *Rouge*, *Bleu* et *Jaune* peuvent être chacune traitée en parallèle.</span><span class="sxs-lookup"><span data-stu-id="96463-111">For example, if an item is configured using *Color*, then variants for *Red*, *Blue*, and *Yellow* could each be processed in parallel.</span></span>

<span data-ttu-id="96463-112">Cela signifie que si le même article avec les mêmes dimensions au-dessus de l’emplacement est en cours de répartition par une vague, d’autres vagues devront attendre pour acquérir un verrou sur le même article et les mêmes dimensions.</span><span class="sxs-lookup"><span data-stu-id="96463-112">This means that if the same item with the same dimensions above the location is being allocated by one wave, other waves will have to wait to acquire a lock on the same item and dimensions.</span></span> <span data-ttu-id="96463-113">S’il n’est pas possible d’acquérir le verrou en temps opportun, une erreur se produira et le traitement de la vague échouera.</span><span class="sxs-lookup"><span data-stu-id="96463-113">If the lock can't be acquired in a timely manner, an error will occur and the wave processing will fail.</span></span>

## <a name="performance-improvements"></a><span data-ttu-id="96463-115">Améliorations des performances</span><span class="sxs-lookup"><span data-stu-id="96463-115">Performance improvements</span></span>

- <span data-ttu-id="96463-117">**Amélioration du débit** : le débit des vagues s’améliorera généralement, même si le traitement parallèle n’est pas configuré, en particulier pour les scénarios où il n’y a pas de chevauchement des articles dans les vagues.</span><span class="sxs-lookup"><span data-stu-id="96463-117">**Improved throughput** - The throughput of waves will typically improve even if parallel processing isn't configured, especially for scenarios where there is no overlap of items within the waves.</span></span>
