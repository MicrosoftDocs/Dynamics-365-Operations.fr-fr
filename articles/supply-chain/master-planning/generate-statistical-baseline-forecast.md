# <a name="generate-a-statistical-baseline-forecast"></a><span data-ttu-id="cc778-103">Générer des prévisions de base statistiques</span><span class="sxs-lookup"><span data-stu-id="cc778-103">Generate a statistical baseline forecast</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="cc778-104">Cette rubrique fournit des informations sur les paramètres et les filtres utilisés dans le calcul de prévision de la demande.</span><span class="sxs-lookup"><span data-stu-id="cc778-104">This topic provides information about the parameters and filters that are used in the calculation of demand forecasting.</span></span>

<span data-ttu-id="cc778-107">Pour générer une prévision de la demande, allez dans **Planification &gt; Prévisions &gt; Prévision de la demande &gt; Générer des prévisions de base statistiques**.</span><span class="sxs-lookup"><span data-stu-id="cc778-107">To generate a demand forecast, go to **Master planning &gt; Forecasting &gt; Demand forecasting &gt; Generate statistical baseline forecast**.</span></span> 

<span data-ttu-id="cc778-108">L’intervalle de prévision peut être sélectionné au moment de la génération de prévision.</span><span class="sxs-lookup"><span data-stu-id="cc778-108">The forecast bucket can be selected at forecast generation time.</span></span> <span data-ttu-id="cc778-109">Les valeurs disponibles sont : jour, semaine et mois.</span><span class="sxs-lookup"><span data-stu-id="cc778-109">The available values are: Day, Week, and Month.</span></span> 

<span data-ttu-id="cc778-110">Le nombre d’intervalles pour lesquels générer une prévision se définit dans le champ **Horizon de prévision**.</span><span class="sxs-lookup"><span data-stu-id="cc778-110">The number of buckets to generate a forecast for is set in the **Forecast horizon** field.</span></span> 

<span data-ttu-id="cc778-111">Lorsque la stratégie de prévision est définie sur **Copier sur la demande historique**, la fin de l’horizon historique est ignorée.</span><span class="sxs-lookup"><span data-stu-id="cc778-111">When the forecast strategy is set to **Copy over historical demand**, the end of the historical horizon is ignored.</span></span> <span data-ttu-id="cc778-112">Le système copie le nombre d’intervalles spécifiés dans le champ **Horizon de prévision** de la demande de prévision, en commençant par la date définie dans le champ **Date de début** sous **Horizon historique**.</span><span class="sxs-lookup"><span data-stu-id="cc778-112">The system copies the number of buckets specified in the **Forecast horizon** field to the forecast demand, starting from the date set in the **From date** field under **Historical horizon**.</span></span> <span data-ttu-id="cc778-113">En copiant une demande historique à partir d’une certaine date, les responsables de production peuvent effectuer le plan pour le trimestre suivant de deux manières :</span><span class="sxs-lookup"><span data-stu-id="cc778-113">By copying historical demand from a certain date forward, production planners can make the plan for the next quarter in two ways:</span></span>

-   <span data-ttu-id="cc778-114">En copiant la demande du même trimestre de l’année dernière.</span><span class="sxs-lookup"><span data-stu-id="cc778-114">By copying the demand from the same quarter last year.</span></span>
-   <span data-ttu-id="cc778-115">En copiant la demande du trimestre précédent.</span><span class="sxs-lookup"><span data-stu-id="cc778-115">By copying the demand from the previous quarter.</span></span>

<span data-ttu-id="cc778-116">Pour empêcher toute confusion dans les calendriers de production, un certain nombre d’intervalles de prévision peuvent être figés.</span><span class="sxs-lookup"><span data-stu-id="cc778-116">To prevent confusion in the production plans, a certain number of forecast buckets can be frozen.</span></span> <span data-ttu-id="cc778-117">Ce nombre est défini dans le champ **Plage de temps bloquée**.</span><span class="sxs-lookup"><span data-stu-id="cc778-117">This number is set in the **Freeze time fence** field.</span></span> <span data-ttu-id="cc778-118">Dans la page **Ajustement de la prévision de demande**, les cellules pour les intervalles figés sont désactivées, pour donner une indication visuelle que ces valeurs ne doivent pas être modifiées.</span><span class="sxs-lookup"><span data-stu-id="cc778-118">On the **Adjusted demand forecast** page, the cells for the frozen buckets are disabled, to give a visual indication that those values should not be changed.</span></span> 

<span data-ttu-id="cc778-119">La date de début de la prévision de la demande de base ne doit pas être la date du jour ou une date dans le futur.</span><span class="sxs-lookup"><span data-stu-id="cc778-119">The start date for the baseline demand forecast doesn’t have to be the current date or a date in the future.</span></span> <span data-ttu-id="cc778-120">Pour définir une date de début différente, utilisez le champ **Date de début de prévision de base - Date de début**.</span><span class="sxs-lookup"><span data-stu-id="cc778-120">To set a different start date, use the **Baseline forecast start date - From date** field.</span></span> <span data-ttu-id="cc778-121">Par exemple, en juin, les utilisateurs peuvent générer une prévision pour l’année suivante.</span><span class="sxs-lookup"><span data-stu-id="cc778-121">For example, in June, users can generate a forecast for the next year.</span></span> <span data-ttu-id="cc778-122">Étant donné que les intervalles de prévision entre la fin d’une demande historique et le début de la base sont manquants, il se peut que les prévisions ne soient pas précises.</span><span class="sxs-lookup"><span data-stu-id="cc778-122">Because the forecast buckets between the end of historical demand and the start of the baseline are missing, the predictions might not be accurate.</span></span> <span data-ttu-id="cc778-123">Si vous utilisez le service de prévision de la demande, il existe quatre méthodes qui permettent de combler les écarts manquants.</span><span class="sxs-lookup"><span data-stu-id="cc778-123">If you are using the Demand forecasting service, there are four ways in which you can fill in the missing gaps.</span></span> <span data-ttu-id="cc778-124">Vous pouvez sélectionner la méthode que vous souhaitez en définissant le paramètre MISSING\_VALUE\_SUBSTITUTION dans la page **Paramètres de prévision de la demande**.</span><span class="sxs-lookup"><span data-stu-id="cc778-124">You can choose the method that you want by setting the MISSING\_VALUE\_SUBSTITUTION parameter on the **Demand forecasting parameters** page.</span></span> 

> [!NOTE]
> <span data-ttu-id="cc778-125">Le remplacement des valeurs manquantes ne fonctionne que pour les lacunes de données entre les dates de début et de fin des données historiques.</span><span class="sxs-lookup"><span data-stu-id="cc778-125">Missing value substitution only works for the gaps in data in between the start and end dates for historical data.</span></span> <span data-ttu-id="cc778-126">Cela ne remplit pas les données avant ou après le dernier point d’information physique. Cela ne procède que par extrapolation entre points d’informations existants réels.</span><span class="sxs-lookup"><span data-stu-id="cc778-126">It will not fill in data before or after the last physical data point, it only acts as extrapolation between actual existing data points.</span></span> 

<span data-ttu-id="cc778-127">Le champ **Date de début de prévision de base** - **Date de début** doit être défini au début d’un intervalle de prévision, par exemple, aux États-unis, un dimanche si l’intervalle de prévision est la semaine.</span><span class="sxs-lookup"><span data-stu-id="cc778-127">The **Baseline forecast start date** - **From date** field has to be set to the beginning of a forecast bucket, for example, in the United States, a Sunday if the forecasting bucket is the week.</span></span> <span data-ttu-id="cc778-128">Le système règle automatiquement le champ **Date de début de prévision de base** - **Date de début** pour correspondre au début d’un intervalle de prévision.</span><span class="sxs-lookup"><span data-stu-id="cc778-128">The system automatically adjusts the **Baseline forecast start date** - **From date** field to match the beginning of a forecast bucket.</span></span> 

<span data-ttu-id="cc778-129">Le champ **Date de début de prévision de base** - **Date de début** peut être réglé sur une date passée.</span><span class="sxs-lookup"><span data-stu-id="cc778-129">The **Baseline forecast start date** - **From date** field can be set to a date in in the past.</span></span> <span data-ttu-id="cc778-130">Autrement dit, il est possible de générer une prévision de la demande dans le passé.</span><span class="sxs-lookup"><span data-stu-id="cc778-130">In other words, it is possible to generate a demand forecast in the past.</span></span> <span data-ttu-id="cc778-131">Cela est utile pour permettre aux utilisateurs d’ajuster les paramètres du service de prévision afin que la prévision statistique générée dans le passé corresponde à la demande réelle historique.</span><span class="sxs-lookup"><span data-stu-id="cc778-131">This is useful, because it lets users adjust the forecast service parameters so that the statistical forecast generated in the past matches the actual historical demand.</span></span> <span data-ttu-id="cc778-132">Les utilisateurs peuvent alors continuer à utiliser ces paramètres pour générer des prévisions de base statistiques pour L’avenir.</span><span class="sxs-lookup"><span data-stu-id="cc778-132">Users can then continue using these parameter settings to generate a statistical baseline forecast for the future.</span></span>

data-ttu-id="cc778-134">Si la case à cocher est désactivée, les ajustements manuels ne sont pas ajoutés à la prévision de base, mais ils ne sont pas supprimés.</span><span class="sxs-lookup"><span data-stu-id="cc778-134">If the check box is cleared, the manual adjustments are not added to the baseline forecast – but they are not deleted.</span></span> <span data-ttu-id="cc778-135">Les ajustements manuels apportés à une prévision peuvent être supprimés uniquement au moment de l’importation de la prévision, en désactivant la case à cocher **Enregistrer les ajustements manuels apportés à la prévision de la demande de base**.</span><span class="sxs-lookup"><span data-stu-id="cc778-135">Manual adjustments made to a forecast can be deleted only at forecast import time, by clearing the **Save the manual adjustments made to the baseline demand forecast** check box.</span></span> <span data-ttu-id="cc778-136">Les ajustements manuels sont enregistrés au moment de l’autorisation.</span><span class="sxs-lookup"><span data-stu-id="cc778-136">Manual adjustments are saved at authorization time.</span></span> <span data-ttu-id="cc778-137">Par conséquent, si un utilisateur effectue des ajustements manuels sur la prévision, mais n’autorise pas la prévision sur Supply Chain Management, les modifications sont perdues.</span><span class="sxs-lookup"><span data-stu-id="cc778-137">Therefore, if a user makes manual adjustments to the forecast, but doesn’t authorize the forecast back to Supply Chain Management, the changes are lost.</span></span> <span class="sxs-lookup"><span data-stu-id="cc778-138">For more information about manual adjustments and how they work, see [Authorize an adjusted forecast](authorize-adjusted-forecast.md).</span></span> 

<span data-ttu-id="cc778-139">Une génération de prévision de la demande peut avoir un nom et des commentaires pour aider les utilisateurs à identifier la prévision générée.</span><span class="sxs-lookup"><span data-stu-id="cc778-139">A demand forecast generation can have a name and comments to help users identify the forecast that has been generated.</span></span> <span data-ttu-id="cc778-140">Ces valeurs sont visibles dans l’historique de génération de prévisions dans la page **Enregistrer les ajustements manuels apportés à la prévision de la demande de base**.</span><span class="sxs-lookup"><span data-stu-id="cc778-140">These values are visible in forecast generation history on the **Statistical baseline forecast generation history** page.</span></span> 

<span data-ttu-id="cc778-141">Le groupe d’organisation intersociétés, les clés de répartition par article, ainsi que d’autres filtres peuvent être appliqués au moment de la génération de prévision.</span><span class="sxs-lookup"><span data-stu-id="cc778-141">The intercompany planning group, item allocation keys, and other filters can be applied at forecast generation time.</span></span> <span data-ttu-id="cc778-142">Ceux-ci peuvent être utilisés pour améliorer les performances ou pour fractionner les données en segments maniables.</span><span class="sxs-lookup"><span data-stu-id="cc778-142">These can be used to improve performance or to split the data into manageable chunks.</span></span> <span data-ttu-id="cc778-143">Toutefois, notez qu’une prévision de la demande n’est pas générée pour les membres d’une clé de répartition par article qui n’est pas associée à un groupe d’organisation intersociétés, même si cette clé de répartition par article est sélectionnée dans la requête.</span><span class="sxs-lookup"><span data-stu-id="cc778-143">However, note that a demand forecast is not generated for the members of any item allocation key that is not associated with an intercompany planning group, even if the item allocation key is selected in the query.</span></span>
