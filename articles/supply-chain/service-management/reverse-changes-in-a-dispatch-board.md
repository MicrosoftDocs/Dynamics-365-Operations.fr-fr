[!include [banner](../includes/banner.md)]


<span data-ttu-id="9b626-104">Cette rubrique décrit la procédure de contrepassation des modifications non enregistrées dans un tableau d'affectation.</span><span class="sxs-lookup"><span data-stu-id="9b626-104">This topic describes how to reverse unsaved modifications that you make in a dispatch board.</span></span> <span data-ttu-id="9b626-105">Par exemple, vous affectez un collaborateur à une activité de service, vous enregistrez l'information, puis vous décidez ultérieurement d'affecter un autre collaborateur à l'activité de service.</span><span class="sxs-lookup"><span data-stu-id="9b626-105">For example, you assign a worker to a service activity, save the information, and then later decide to assign a different worker to the service activity.</span></span> <span data-ttu-id="9b626-106">Vous modifiez alors le collaborateur dans le tableau d'affectation, mais avant d'enregistrer la modification, vous apprenez que le collaborateur que vous venez d'affecter n'est pas disponible.</span><span class="sxs-lookup"><span data-stu-id="9b626-106">You modify the worker in the dispatch board, and then, before saving the change, learn that the worker just assigned is not available.</span></span> <span data-ttu-id="9b626-107">Vous pouvez contrepasser la modification non enregistrée afin que le collaborateur d'origine soit affecté à nouveau à la commande de service.</span><span class="sxs-lookup"><span data-stu-id="9b626-107">You can reverse the unsaved modification so that the original worker is reassigned to the service order.</span></span>

<span data-ttu-id="9b626-108">Utilisez les étapes suivantes pour contrepasser les modifications non enregistrées dans un tableau d'affectation :</span><span class="sxs-lookup"><span data-stu-id="9b626-108">Use the following steps to reverse unsaved changes in a dispatch board:</span></span>
