# <a name="manage-standard-cost-updates"></a><span data-ttu-id="cf4f2-103">Gérer des mises à jour des coûts standard</span><span class="sxs-lookup"><span data-stu-id="cf4f2-103">Manage standard cost updates</span></span>

[!include[banner](../includes/banner.md)]


<span data-ttu-id="cf4f2-104">Les mises à jour des données de coût standard peuvent être gérées à l'aide de deux approches : l'approche à une version ou l'approche à deux versions.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-104">Updates to standard cost data can be managed by using two different approaches - the one-version approach or the two-version approach.</span></span> 

<span data-ttu-id="cf4f2-105">La première approche utilise une seule version d'évaluation des coûts qui contient tous les enregistrements de coûts.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-105">The one-version approach uses a single costing version that contains all cost records.</span></span> <span data-ttu-id="cf4f2-106">Ces enregistrements incluent les coûts d'origine et les mises à jour des coûts.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-106">These records include the original costs and all cost updates.</span></span>

<span data-ttu-id="cf4f2-107">La deuxième approche utilise une version qui contient les enregistrements des coûts originaux et une autre version qui contient les enregistrements de toutes les mises à jour des coûts.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-107">The two-version approach uses one version that contains records of the original costs and a second version that contains records of all cost updates.</span></span> <span data-ttu-id="cf4f2-108">Elle présente l'avantage de délimiter et suivre clairement les mises à jour des coûts au sein d'une version d'évaluation des coûts distincte, sans affecter la version d'évaluation des coûts d'origine.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-108">A primary advantage of the two-version approach is the clear delineation and tracking of cost updates in a separate costing version, without affecting the original costing version.</span></span> <span data-ttu-id="cf4f2-109">Elle peut être utilisée pour identifier plusieurs mises à jour incrémentielles, chaque mise à jour incrémentielle ayant une version d'évaluation des coûts distincte qui contient les enregistrements de coûts incrémentiels.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-109">The two-version approach can be used to identify multiple incremental updates, where each incremental update has a separate costing version that contains the incremental cost records.</span></span>

<span data-ttu-id="cf4f2-111">L'exemple suivant illustre l'utilisation des approches à une version et à deux versions pour mettre à jour les coûts standard dans un environnement de year.</span></span> <span data-ttu-id="cf4f2-114">L'identificateur de cette version est 2016-STD.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-114">The identifier for this version is 2016-STD.</span></span> <span data-ttu-id="cf4f2-115">La version 2016-STD contient les coûts actifs actuels pour tous les articles.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-115">Version -   <span data-ttu-id="cf4f2-118">**Approche à une version des mises à jour de données de coûts** − Dans l'approche à une version, la version d'évaluation des coûts d'origine 2016-STD contient tous les enregistrements de coûts.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-118">**One-version approach to cost data updates** − In the one-version approach, the original costing version 2016-STD contains all cost records.</span></span> <span data-ttu-id="cf4f2-119">Les mises à jour des coûts sont enregistrées dans 2016-STD et sont définies sur le statut « En attente ».</span><span class="sxs-lookup"><span data-stu-id="cf4f2-119">Cost updates are recorded in 2016-STD and are set to a status of ”Pending.”</span></span> <span data-ttu-id="cf4f2-120">Les coûts en cours peuvent être entrés manuellement pour les nouveaux articles ou être calculés pour un article fabriqué pour refléter les corrections.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-120">The pending costs can be manually entered for new purchased items, or they can be calculated for a manufactured item to reflect corrections.</span></span> <span data-ttu-id="cf4f2-121">Lorsque l'approche à une version est utilisée, les calculs de nomenclature ne nécessitent pas de source de données de secours car tous les coûts actifs sont contenus dans la version d'évaluation des coûts.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-121">When the one-version approach is used, the BOM calculations do not require a fallback data source because all active costs are contained in the costing version.</span></span> <span data-ttu-id="cf4f2-122">Une fois les coûts en cours activés, la version d'évaluation des coûts d'origine 2016-STD contient de nouveau tous les coûts actifs actuels.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-122">After the pending costs become active, the original costing version 2016-STD will again contain all the current active costs.</span></span>
-   <span data-ttu-id="cf4f2-123">**Approche à deux versions des mises à jour de données de coûts** − L'approche à deux versions nécessite une version d'évaluation des coûts supplémentaire qui data-stu-id="cf4f2-124">The identifier for this version is 2016-STD-CHANGES.</span></span> <span data-ttu-id="cf4f2-125">Les mises à jour des coûts sont enregistrées dans 2016-STD-CHANGES et sont définies sur le statut « En attente ».</span><span class="sxs-lookup"><span data-stu-id="cf4f2-125">Cost updates are recorded in 2016-STD-CHANGES and are set to a status of “Pending.”</span></span> <span data-ttu-id="cf4f2-126">Avec l'approche à deux versions, les calculs de nomenclature des coûts en attente pour les articles fabriqués nécessitent une source de données de secours.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-128">The fallback can be expressed as the active costs or as the costing version 2016-STD, because both identify the source of cost data when it is not data-ttu-id="cf4f2-130">Lorsque l'approche à deux versions est utilisée, les stratégies de blocage pour la version d'évaluation des coûts d'origine doivent être définies de façon à empêcher les mises début spécifiée doit être mise à jour avec chaque lot de modifications pour refléter la date d'activation prévue.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-132">The specified from-date should be updated with each batch of changes to reflect the scheduled activation date.</span></span>

<span data-ttu-id="cf4f2-133">L'exemple utilise une version d'évaluation des coûts supplémentaire pour la gestion des mises à jour au cours de l'année 2016.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-133">This example used one additional costing version for managing updates throughout the year 2016.</span></span> <span data-ttu-id="cf4f2-134">Plusieurs versions d'évaluation des coûts supplémentaires peuvent être utilisées, telles qu'une version distincte pour chaque lot de mises à jour.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-134">More than one additional costing version can be used, such as a separate version for each batch of updates.</span></span> <span data-ttu-id="cf4f2-135">Dans ce cas, le secours doit être exprimé comme les coûts actifs car ils pourraient être dispersés entre plusieurs versions d'évaluation des coûts.</span><span class="sxs-lookup"><span data-stu-id="cf4f2-135">When more than one additional costing is used, the fallback must be expressed as the active costs, because the active costs are spread over multiple costing versions.</span></span>
